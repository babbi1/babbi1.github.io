将 Hexo 博客部署到 GitHub Pages ( .github.io 项目) 也是一个非常常见的方案。正如我们之前讨论的，GitHub Pages 非常适合托管来自 GitHub 仓库的静态网站，并且对于公共仓库是 无费用 的。

Hexo 内置了方便的部署功能，可以很轻松地将生成的静态文件推送到 GitHub Pages 仓库。

以下是将您的 Hexo 博客部署到 GitHub Pages 的详细步骤：

步骤 1：在 GitHub 上创建仓库

  • 重要： GitHub Pages 对仓库名称有特殊要求，特别是对于用户或组织的个人主页网站。

    • 如果您想创建一个 个人主页网站 （网址是 您的GitHub用户名.github.io ），您需要创建一个名为 您的GitHub用户名.github.io 的新仓库。例如，如果您的 GitHub 用户名是 octocat ，则仓库名称必须是 octocat.github.io 。 这个仓库必须是公共的 (Public)。

    • 如果您想创建一个 项目网站 （网址通常是 您的GitHub用户名.github.io/仓库名称 ），您可以在任何仓库中启用 GitHub Pages，通常是将静态文件放在 main 或 gh-pages 分支的根目录或 /docs 文件夹中。但对于博客，通常会创建个人主页网站仓库。

  • 怎么做：

    1. 登录您的 GitHub 账号。

    2. 点击右上角的加号 + ，选择 “New repository” (新建仓库)。

    3. Repository name (仓库名称): 输入 您的GitHub用户名.github.io (替换为您的实际用户名)。

    4. Description (描述): 可以选择性地添加描述。

    5. Public/Private: 务必选择 Public (公共) ，GitHub Pages 不支持私有仓库的个人主页网站。

    6. 不要勾选 “Add a README file” 或其他选项，创建一个完全空的仓库。

    7. 点击 “Create repository” (创建仓库)。

步骤 2：在本地 Hexo 项目中安装 Git 部署插件

  • Hexo 默认不包含部署到 Git 仓库的功能，您需要安装一个插件。

  • 怎么做： 打开命令行终端，进入您的 Hexo 项目根目录，运行以下命令安装 hexo-deployer-git 插件：

    npm install hexo-deployer-git –save

步骤 3：配置 Hexo 的部署设置

  • 您需要告诉 Hexo 将生成的静态文件部署到哪个 Git 仓库。

  • 怎么做： 打开您的项目根目录下的主配置文件 _config.yml 。找到或添加 deploy 部分。

    deploy:  type: git

      repo: https://github.com/您的GitHub用户名/您的GitHub用户名.github.io.git  branch: main # 或者 master，取决于您的 GitHub 仓库默认分支

    content_copy

    展开

    • 将 您的GitHub用户名 替换为您的实际 GitHub 用户名。

    • repo: 后面填写您刚刚创建的 GitHub Pages 仓库的 HTTPS 克隆地址。

    • branch: 通常是 main 或 master 。请查看您的 GitHub 仓库设置，确认默认分支名称。Hexo 会将生成的静态文件推送到这个分支的根目录。

步骤 4：生成静态文件并部署到 GitHub Pages

  • 现在，您可以使用 Hexo 的命令来生成静态文件，并直接将其部署到您配置好的 GitHub Pages 仓库。

  • 怎么做：

    1. 打开命令行终端，进入您的 Hexo 项目根目录。

    2. 生成静态文件： 运行 hexo generate 或 hexo g 。这个步骤会将您的博客生成到 public 文件夹。

    3. 部署到 GitHub Pages： 运行 hexo deploy 或简写 hexo d 。

      • 这个命令会自动将 public 文件夹中的内容推送到您在 _config.yml 中配置的 Git 仓库 ( 您的GitHub用户名.github.io.git ) 的指定分支 ( main 或 master )。

      • 首次部署时，可能需要您输入 GitHub 的用户名和密码，或者使用 Personal Access Token (PAT) 进行认证。推荐使用 PAT，更安全。您可以参考 GitHub 文档如何生成 PAT。

  • 等待部署： 推送完成后，GitHub Pages 可能需要几分钟的时间来处理和部署您的网站。

步骤 5：访问您的 GitHub Pages 博客

  • 部署成功并在 GitHub Pages 处理完成后，您就可以通过浏览器访问您的博客了。

  • 网址： https://您的GitHub用户名.github.io (替换为您的实际用户名)。

总结流程：

  1. 在 GitHub 创建 您的GitHub用户名.github.io 公共仓库。

  2. 在本地 Hexo 项目安装 hexo-deployer-git ( npm install hexo-deployer-git --save )。

  3. 修改 _config.yml ，配置 deploy 部分，指定仓库地址和分支。

  4. 运行 hexo generate ( hexo g ) 生成静态文件。

  5. 运行 hexo deploy ( hexo d ) 部署到 GitHub Pages。

  6. 等待几分钟，访问 https://您的GitHub用户名.github.io 。

这就是将 Hexo 博客部署到 GitHub Pages 的方法。与 Firebase Hosting 不同，这种方式直接利用了 GitHub 的仓库和 Pages 服务进行托管。

您可以根据您的需求（比如是否需要 Firebase 的其他服务集成、对性能和 CDN 的更高要求等）来选择是部署到 Firebase Hosting 还是 GitHub Pages。

希望这份指南对您有帮助！开始您的 GitHub Pages 部署之旅吧！
