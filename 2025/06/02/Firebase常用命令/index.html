Firebase CLI 就像您的一个万能工具箱，里面有很多不同的工具（命令），每个都用于处理 Firebase 的某个特定功能。

这里列出一些您最常用、最有用的命令：

  1. firebase init : 这是您开始一个新项目或者将 Firebase 添加到现有项目时最先使用的命令。它会帮助您设置项目文件，并选择您想要使用的 Firebase 产品（比如 Hosting, Cloud Functions, Firestore 等等）。

  2. firebase deploy : 这是用来将您的本地代码或资源（比如网站文件、Cloud Functions 函数、安全规则等）部署到您的 Firebase 项目的云端服务上的命令。用它来发布您的应用！

  3. firebase serve : 主要用于 Firebase Hosting 和 Cloud Functions。它会在您本地启动一个服务器，让您可以在部署之前测试您的网站或函数。

  4. firebase emulators:start : 这个非常有用！它可以在您本地启动 Firebase 模拟器套件，模拟各种 Firebase 产品（Authentication, Firestore, Realtime Database, Functions, Hosting 等），这样您就可以在本地进行开发和测试，而无需连接到真实的云端项目，而且通常是 无费用 的！

  5. firebase projects:list : 查看您关联的 Google 账号下有哪些 Firebase 项目。

  6. firebase use [项目ID] : 切换到您想要操作的特定 Firebase 项目。您当前的项目 ID 是 tsmz-4920f 。

  7. 还有其他命令用于管理特定产品，比如 Cloud Firestore 的索引、Realtime Database 的数据导入导出、Cloud Storage 的文件等等。

这些只是冰山一角！CLI 还有很多其他命令。您随时可以在命令行输入 firebase --help 来查看所有可用的命令列表，或者输入 firebase [命令名称] --help 来查看某个特定命令的详细用法。

现在您已经登录并了解了一些基本命令，是不是想试试看用 firebase init 初始化您的项目，或者直接用 firebase emulators:start
