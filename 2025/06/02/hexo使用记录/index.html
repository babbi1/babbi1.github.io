source文件夹相当于根目录
source/_posts/ 文章
source/images/ 图片

  • 理念： Hexo 提供一个“资源文件夹”功能。开启后，当您通过 hexo new "文章标题" 创建一篇新文章时，Hexo 会自动在 source/_posts/ 文件夹下创建一个 与文章同名 的文件夹。这个文件夹就是该文章专属的资源文件夹。您可以将这篇文章相关的图片或其他资源文件直接放到这个同名文件夹里。

  • 怎么做：

    1. 开启功能： 打开您的项目根目录下的主配置文件 _config.yml 。找到 post_asset_folder: 这一行，将其值设置为 true 。

      post_asset_folder: true

      content_copy

    2. 重新创建文章或手动创建文件夹： 开启功能后，下次您运行 hexo new "新文章" 时，Hexo 就会自动创建 /source/_posts/新文章/ 文件夹。如果您想为已有的文章开启此功能，需要手动在 source/_posts/ 下创建与文章 .md 文件同名的文件夹。

    3. 将图片放入资源文件夹： 将该篇文章相关的图片文件复制到这个 同名资源文件夹 中（例如 /source/_posts/新文章/photo.jpg ）。

    4. 在文章中引用图片： 在您的文章 Markdown 文件 ( source/_posts/新文章.md ) 中，使用一个特殊的标签插件来引用资源文件夹里的图片。Hexo 默认提供了 asset_img 标签。

      也可以添加 alt 文本

      content_copy

  • 优点：

    • 图片与文章文件紧密关联，删除文章时，其资源文件夹也可以一并删除，管理更方便，不容易产生“孤立”图片。

    • 对于管理大量文章和图片非常有效。

  • 缺点：

    • 引用图片的语法稍有不同 ( {% asset_img ... %} 标签而不是标准的 Markdown ![]() )。

    • 图片分散在各个文章的资源文件夹中，不够集中。

在 3-hexo 主题的博客文章底部添加一段文字，说明“转载请注明来源”、“欢迎考证和指出错误”、“可以在下面评论区评论，也可以邮件至 jaytp@qq.com  ”等内容。

这段文字通常被称为“ 文章底部声明 ”或“ 版权信息 ”。在 Hexo 的主题中，这类固定的、出现在每篇文章相同位置的内容，通常是通过修改 主题的模板文件 来实现的。

在 3-hexo 主题中，控制文章内容区域显示的是一个模板文件。您需要找到并修改这个文件。

根据 Hexo 和 3-hexo 主题的常见结构，这段内容最有可能位于：

  1. 文章内容模板文件： 负责渲染单篇博客文章内容的布局文件。它会包含文章标题、作者、日期、正文，以及可能在正文之后、评论区之前的内容。

  2. 文章布局文件： 有些主题会将文章内容包含在一个更大的布局文件中。

在 3-hexo 主题中，您很可能需要修改位于 themes/3-hexo/layout/_partial/article.ejs 这个文件。

为什么是这个文件？

  • themes/3-hexo/ : 这是您的 3-hexo 主题的根目录。

  • layout/ : 这个文件夹通常存放主题的布局文件和模板文件。

  • _partial/ : 这个文件夹通常存放一些“局部”或“片段”的模板文件，这些文件会在其他主布局文件中被引用。

  • article.ejs : 这是一个使用 EJS 模板引擎编写的文件，其文件名 article 强烈暗示它负责渲染单篇博客文章的内容区域。

如何操作：

  1. 打开您的 Hexo 项目文件夹。

  2. 导航到 themes/3-hexo/layout/_partial/ 文件夹。

  3. 使用文本编辑器打开 article.ejs 文件。

  4. 仔细阅读这个文件。您会看到一些 HTML 结构和 EJS 模板代码（以 <% 和 %> 开头）。

  5. 找到渲染文章正文的部分。通常会看到类似 <%- post.content %> 或类似的 EJS 代码，这是 Hexo 将您 Markdown 文章渲染后的 HTML 内容插入的位置。

  6. 在 <%- post.content %> 之后 ，但通常在评论区代码 之前 ，添加您的那段文字。您可以将其放在一个 <p> 标签或 <div> 标签中，并添加一些样式（比如 <div class="post-copyright"> ）。

  7. 保存 article.ejs 文件。

  8. 运行 hexo clean 和 hexo generate ( hexo cl && hexo g ) 来清理缓存并重新生成静态文件，使您的修改生效。

  9. 运行 hexo server 或 firebase emulators:start 在本地预览，确认您添加的文字已经正确显示在文章底部了。

  10. 如果满意，运行 firebase deploy 部署到线上。

重要提示 (关于直接修改主题文件):

直接修改 themes/3-hexo/ 文件夹里面的文件有一个 缺点 ：当 3-hexo 主题发布新版本时，您通过 npm 或 Git 更新主题，您的修改可能会被 覆盖掉 ！

为了避免这个问题，Hexo 提供了 主题覆盖 (Theme Overrides) 的机制。推荐的做法是：

  1. 在您的项目根目录下，创建一个与主题模板文件相同路径的文件夹结构： layout/_partial/ 。

  2. 将 themes/3-hexo/layout/_partial/article.ejs 这个文件 复制 到您项目根目录下的 layout/_partial/ 文件夹中。

  3. 在您项目根目录下的 layout/_partial/article.ejs 文件中进行修改。

这样，Hexo 在生成静态文件时，会优先使用您项目根目录下的 layout/_partial/article.ejs 文件，而忽略主题文件夹中的同名文件。即使您更新了主题，您的自定义修改也会被保留。

所以，更推荐的步骤是：

  1. 在项目根目录创建 layout/_partial/ 文件夹。

  2. 复制 themes/3-hexo/layout/_partial/article.ejs 到项目根目录的 layout/_partial/ 。

  3. 修改项目根目录下的 layout/_partial/article.ejs 文件，添加您的声明文字。

  4. 保存并运行 hexo cl && hexo g 和 firebase deploy 。

查找模板文件可能需要您对主题的结构和模板引擎（EJS 在这个例子中）有一些了解。但根据您的描述， themes/3-hexo/layout/_partial/article.ejs 是最有可能是您需要修改的文件。
