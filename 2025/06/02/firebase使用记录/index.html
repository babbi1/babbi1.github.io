安装node.js

https://nodejs.org/
验证

安装firebase cli工具

npm install -g firebase-tools
安装完成后，验证是否安装成功：
firebase –version

获取授权地址：

firebase login –no-localhost

初始化项目:

firebase init hosting

运行 Firebase 模拟器命令
firebase emulators:start
当您访问 http://127.0.0.1:5000 时，Hosting 模拟器就会从刚刚生成的、包含了您博客内容的 public 文件夹中提供文件了，您应该就能看到您的博客了！

创建链接 /发布
firebase deploy
只展示项目根目录public文件夹下内容

使用hexo

创建项目
mkdir myblog
cd myblog
hexo init
cp ../firebase/firebase.json ./
hexo new hello-test
hexo g # 编辑后使用。生成 public 文件夹及其里面的所有静态文件
hexo clean # 删除文章后使用。删除 Hexo 生成的 public 文件夹以及 Hexo 的缓存文件
hexo g # 编辑后使用。生成 public 文件夹及其里面的所有静态文件

复制后设置默认项目

  • 使用 firebase use 命令将您的 test-0000f 项目设置为当前文件夹的默认项目。这样以后在这个文件夹里运行 Firebase 命令时，CLI 会自动知道是针对哪个项目。

  • 怎么做：

    1. 打开命令行终端，进入您的 Hexo 项目根目录。

    2. 运行命令： firebase use --add

    3. CLI 会列出您账号下所有的 Firebase 项目。找到您的 test-0000f 项目，并根据提示输入对应的编号或项目 ID 来选择它。

    4. CLI 可能会问您是否要为此项目设置一个别名 (alias)，您可以输入一个简短易记的名字，比如 default 或者 blog ，或者直接回车使用项目 ID 作为别名。

    5. 设置成功后，CLI 会显示类似 Now using project test-0000f 的信息。

  • 优点： 设置一次后，在这个文件夹里运行其他 Firebase 命令（如 firebase deploy ）时就无需再指定项目了，非常方便。

  • 缺点： 需要额外的一步设置
